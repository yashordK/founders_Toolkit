# Judging Alignment Notes — Founder's Second Brain

Why this project scores well:

| Criterion | How this project addresses it |
|---|---|
| Potential Impact | Solves a real, constant founder pain (forgotten contacts, lost meeting context, no continuity across AI tools) |
| Creativity & Innovation | Graph-based warm-intro reasoning + proactive daily planning is not a generic chatbot-with-memory demo |
| Technical Excellence | Self-hosted Postgres-backed Cognee, custom skills, custom ranking/synthesis logic, locked-down security posture |
| Best Use of Cognee | All four lifecycle verbs used with real purpose: remember() (3 capture channels), recall() (multi-hop + ad-hoc), improve() (feedback-driven reweighting), forget() (scheduled, surgical pruning) — not bolted on for scoring |
| User Experience | One-message capture from a phone; dashboard as a clean visual home; near-zero friction for the core "add a contact" action |
| Presentation Quality | Clear problem → solution → impact narrative; live phone demo as the centerpiece moment |

**Track decision:** Best Use of Open Source, not Cognee Cloud. Self-hosting demonstrates deeper technical engagement, avoids demo dependency on an external service's uptime, and the open-source track is likely less saturated than the low-friction cloud signup path.
